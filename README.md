# pytest_ui_api_template# Документация проекта по автоматизации тестирования

## Описание проекта

Проект представляет собой набор автоматизированных тестов для проверки функциональности веб-приложения

## Описание базового синтаксиса записи и форматирования

Проект использует следующие технологии и синтаксис:

1. **Python**: Основной язык программирования для написания тестов.
2. **Selenium**: Библиотека для автоматизации взаимодействия с веб-браузером.
3. **Pytest**: Фреймворк для написания и запуска тестов.
4. **Allure**: Инструмент для генерации отчетов о выполнении тестов.
5. **Page Object**: Шаблон проектирования (паттерн) в автоматизированном тестировании, который представляет веб-страницу или ее часть как объект класса

### Форматирование кода

- Код форматируется в соответствии с PEP 8 (стиль написания кода на Python).
- Используются docstrings для документирования методов и функций.
- Все шаги теста размечаются с помощью `@allure.step` или `with allure.step` для улучшения читаемости отчетов.

## Режимы запусков автотестов :

Доступные режимы тестирования
Проект поддерживает три режима запуска тестов:

1. UI-тесты — автоматизация через пользовательский интерфейс
	Запуск UI-тестов
   ```bash
   pytest -m ui
   ```

2. API-тесты — автоматизация через REST API
	Запуск API-тестов
   ```bash
   pytest -m api
   ```


3. Все тесты — запуск всех доступных тестов
	Запуск всех тестов
   ```bash
   pytest
   ```

4. Запуск с покрытием
	Запуск всех тестов
   ```bash
   pytest --cov=tests
   ```


## Инструкция по запуску тестов для формирования отчета Allure

   ```

1. Запустите тесты с генерацией отчета Allure:
   ```bash
   pytest --alluredir=./allure-results
   ```

   Эта команда запустит все тесты и сохранит результаты в директорию `./allure-results`.

---

## Инструкция по просмотру сформированного отчета Allure

1. После завершения тестов, чтобы просмотреть отчет, выполните команду:
   ```bash
   allure serve ./allure-results
   ```

   Эта команда запустит локальный сервер и откроет отчет в браузере.

2. В отчете Allure вы увидите:
   - **Название теста**: Например, `Тест полного процесса покупки`.
   - **Описание теста**: Описание, что тест проверяет процесс оформления заказа от поиска модели до оформления заказа.
   - **Шаги теста**:
     - Открытие страницы магазина.
     - Выбор региона.
     - Принятие куки.
     - Поиск товара.
     - Добавление товара в корзину.
     - Оформление товара
     - Оформление самовывоза
     - Заполнение данных получателя.
     - Проверка оформления заказа
   - **Результат**: Успешное выполнение оформления заказа.
